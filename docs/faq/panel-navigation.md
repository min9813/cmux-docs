# パネル・サイドバーのキーボードナビゲーション

## Q: 左パネルにショートカットで移動するには？

**A:** 分割ペイン間のフォーカス移動には以下のショートカットが使えます。

| 方向 | ショートカット |
|------|---------------|
| 左 | `Cmd + Option + ←` |
| 右 | `Cmd + Option + →` |
| 上 | `Cmd + Option + ↑` |
| 下 | `Cmd + Option + ↓` |

これらは Settings のキーボードショートカット設定からカスタマイズ可能です。

## Q: 一番左のサイドバー（ワークスペースリスト）にフォーカスを移動するショートカットは？

**A:** 現時点ではサイドバーに直接キーボードフォーカスを移す専用ショートカットは存在しません。代替手段として以下が使えます。

| 操作 | ショートカット |
|------|---------------|
| サイドバーの表示/非表示 | `Cmd + B` |
| 次のワークスペース | `Ctrl + Cmd + ]` |
| 前のワークスペース | `Ctrl + Cmd + [` |
| ワークスペースに移動（検索） | `Cmd + P` |

## Q: `Cmd+Ctrl+[` と `Cmd+Ctrl+]` のどちらを押しても同じ方向にしか移動しない

**A:** JIS キーボードで発生する既知の問題です。JIS 配列では `[` と `]` の物理キー位置が US 配列と異なるため、修飾キーとの組み合わせで意図しないキーイベントが送られることがあります。

Settings > Keyboard Shortcuts で **Previous Workspace** と **Next Workspace** を別のキーに変更してください。例:

| 操作 | 変更後のショートカット例 |
|------|------------------------|
| 次のワークスペース（下） | `Cmd + Ctrl + ↓` |
| 前のワークスペース（上） | `Cmd + Ctrl + ↑` |
