# SSH 接続中に Mac をスリープした場合の挙動

## Q: cmux で SSH したまま Mac を閉じる（スリープ）とどうなるか？iTerm2 だとパネルがフリーズするが

**A:** cmux では SSH 接続が切れると**自動的に再接続を試みます**。iTerm2 のようにパネルがフリーズすることはありません。

### スリープ → 復帰時の動作

1. Mac がスリープすると SSH トンネルが切断される
2. ワークスペースの接続状態が `connected` → `disconnected` に変わる
3. サイドバーに切断状態が表示される
4. Mac 復帰時に `NSWorkspace.didWakeNotification` を検知し、自動再接続が開始される（指数バックオフ付きリトライ）
5. 再接続成功 → `connected` に戻り、リモートの tmux セッションにそのまま復帰

### iTerm2 との比較

| 項目 | iTerm2 | cmux |
|------|--------|------|
| SSH切断時 | パネルがフリーズ | 切断状態を表示 |
| 再接続 | 手動（新しいセッションを開く必要あり） | 自動（リトライ付き） |
| tmux セッション | 別途 `tmux attach` が必要 | 自動復帰 |
| ControlMaster | なし | SSH多重化で高速再接続 |

### リモート側のプロセスはどうなるか？

リモート側で tmux を使っていれば、プロセス（Claude Code 等）はサーバー上で生き続けています。cmux が再接続した後、そのまま作業を継続できます。

tmux を使っていない場合は、SSH 切断時にリモートのシェルとプロセスが終了します（これは iTerm2 でも同じ挙動です）。
